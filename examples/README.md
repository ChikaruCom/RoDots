# RoDots sample documents

This folder contains small `.rdot` documents for checking the main RoDots workflows.

- `minutes.rdot`: meeting notes with date aliases and row repeat.
- `report.rdot`: report template with file-template friendly metadata.
- `project-kickoff.rdot`: kickoff checklist for editors.
- `daily-handoff.view.rdot`: operator-facing handoff sheet that starts in View mode.
- `operator.view.rdot`: simple operator checklist that starts in View mode.
- `operator.rock.rdot`: locked operator procedure that cannot switch out of View mode.
- `locked-procedure.rock.rdot`: locked approval flow for documents that should be changed only by an owner.
- `【仕様書】共通システム開発_(v1.0.0).rdot`: file-template style output sample.

Open `*.view.rdot` when you want the user to execute a prepared document without seeing source first.
Open `*.rock.rdot` when the document is distributed as a fixed procedure and edits should go through a manager.
